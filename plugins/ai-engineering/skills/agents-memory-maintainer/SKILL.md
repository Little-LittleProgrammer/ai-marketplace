# AGENTS.md 维护

## 概述

此 Skill 用于维护 AGENTS.md 文件，更新项目级别的知识和经验。

## AGENTS.md 结构

```markdown
# 项目名称

## 项目概述
[项目简介、目标、范围]

## 技术栈
[使用的技术栈说明]

## 架构概览
[系统架构概述]

## 代码模式
[常用的代码模式和约定]

### 模式1: {模式名称}
[模式描述和使用场景]

## 最佳实践
[项目特定的最佳实践]

### 实践1: {实践名称}
[实践描述]

## 历史决策
[重要的技术决策记录]

### 决策1: {决策名称}
- **日期**: YYYY-MM-DD
- **背景**: [决策背景]
- **决定**: [决策内容]
- **理由**: [决策理由]

## 常见问题
[常见问题及解决方案]

### 问题1: {问题描述}
**解决方案**: [解决方案]
```

## 更新时机

### 项目启动时
- 创建初始 AGENTS.md
- 填写项目基本信息

### 开发过程中
- 记录重要技术决策
- 更新代码模式
- 添加最佳实践

### 项目完成时
- 总结项目经验
- 提取可复用模式
- 记录教训

## 维护流程

### 1. 收集信息
- 项目开发过程中的知识
- 问题解决方案
- 技术决策

### 2. 整理归纳
- 分类整理知识
- 提取通用模式
- 总结经验教训

### 3. 更新文档
- 更新对应章节
- 保持结构一致
- 确保内容准确

## 最佳实践

1. **及时更新**: 开发过程中及时记录
2. **结构清晰**: 保持文档结构清晰
3. **内容精炼**: 只保留重要信息
4. **定期审核**: 定期审核和清理过时内容
