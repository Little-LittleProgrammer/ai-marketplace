# 方案变更

## 概述

此 Skill 用于处理设计变更，包括架构调整、API修改、数据模型更新等。

## 变更流程

### 1. 变更请求分析

- 理解变更原因
- 分析变更范围
- 评估影响

### 2. 影响分析

评估变更对以下方面的影响：
- 系统架构
- API 兼容性
- 数据迁移
- 已开发代码

### 3. 更新设计文档

- 更新 `design-{id}.md`
- 记录变更历史
- 更新相关图表

### 4. 通知下游

- 通知开发团队
- 更新任务分解
- 调整开发计划

## 变更记录格式

```markdown
## 设计变更历史

| 版本 | 日期 | 变更内容 | 原因 |
|------|------|----------|------|
| 1.1 | 2026-03-16 | 新增缓存层 | 性能优化 |
```

## 变更类型

### 架构变更
- 新增/删除服务
- 调整服务边界
- 修改通信方式

### API 变更
- 新增接口
- 修改接口参数
- 废弃接口

### 数据模型变更
- 新增表/字段
- 修改字段类型
- 数据迁移

## 最佳实践

1. **版本控制**: 保留历史版本
2. **影响评估**: 全面评估变更影响
3. **向后兼容**: 尽量保持向后兼容
4. **文档同步**: 同步更新所有相关文档
