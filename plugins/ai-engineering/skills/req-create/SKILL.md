# 创建需求

## 概述

此 Skill 用于创建新的需求文档，包括需求分析、拆分、边界定义和用户故事编写。

## 使用流程

### 1. 收集需求信息

通过提问收集以下信息：
- 需求背景和目标
- 核心功能点
- 目标用户
- 使用场景
- 技术约束

### 2. 参考记忆系统

- 读取 `AGENTS.md` 了解项目背景
- 检索 `context/patterns/` 中的类似需求模式
- 参考 `context/templates/` 中的需求模板

### 3. 生成需求文档

根据收集的信息生成 `spec-{id}.md`

## 输出模板

```markdown
# 需求规格：{需求名称}

## 概述
[需求背景、目标、范围]

## 功能需求

### {功能模块1}

#### 功能点1.1
- **描述**: [功能描述]
- **输入**: [输入说明]
- **输出**: [输出说明]
- **验收标准**:
  - [ ] 标准1
  - [ ] 标准2

## 非功能需求

- **性能**: [性能要求]
- **安全**: [安全要求]
- **可用性**: [可用性要求]
- **兼容性**: [兼容性要求]

## 边界约束

### 技术约束
- [技术限制]

### 业务约束
- [业务限制]

## 用户故事

### 故事1
作为 [角色]，我希望 [功能]，以便 [价值]。

**验收标准**:
- Given [前置条件]
- When [操作]
- Then [预期结果]

## 附录

[补充信息、参考资料]
```

## 最佳实践

1. **明确边界**: 清晰定义需求范围，避免模糊
2. **可测试**: 验收标准应可测试验证
3. **用户视角**: 从用户角度描述需求
4. **参考历史**: 利用记忆系统中的历史经验
